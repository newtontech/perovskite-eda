# 目标变量分析：Delta_PCE vs PCE（基线区分）

> 生成时间：2026-04-30
> 关键问题：当前项目预测的是 Delta_PCE，但文献主流预测的是绝对 PCE。基线 PCE（without_modulator）的处理方式直接影响模型性能。

---

## 1. 当前项目的目标变量定义

### 1.1 Delta_PCE 的计算方式

```python
Delta_PCE = jv_reverse_scan_pce - jv_reverse_scan_pce_without_modulator
```

- `jv_reverse_scan_pce_without_modulator`：**基线PCE**（无添加剂/钝化剂时的器件效率）
- `jv_reverse_scan_pce`：**处理后PCE**（有添加剂/钝化剂时的器件效率）
- `Delta_PCE`：**效率提升量**（添加剂带来的绝对PCE变化）

### 1.2 数据统计（清洗后 4,172 条）

| 变量 | 均值 | 标准差 | 最小值 | 最大值 | 物理意义 |
|------|------|--------|--------|--------|----------|
| **PCE_without_modulator** | 11.83% | 5.92% | 0% | 30% | 基线器件效率 |
| **PCE_with_modulator** | 14.01% | 5.89% | 0% | 30% | 处理后器件效率 |
| **Delta_PCE** | 2.10% | 3.31% | -19.19% | +24.2% | 效率变化量 |
| **rel_pce_improvement** | — | — | — | — | 相对改善百分比 |

### 1.3 Delta_PCE 的分布特征

- **正数据**：Delta_PCE > 0（添加剂有效，占大多数）
- **负数据**：Delta_PCE < 0（添加剂反而降低性能，占一定比例）
- **极端值**：Delta_PCE = -19.19% 到 +24.2%，范围极宽
- **信噪比低**：差分变量放大测量误差，两个PCE的独立测量误差叠加

## 2. 文献中的目标变量对比

| 文献 | 预测目标 | R² | 说明 |
|------|----------|-----|------|
| **Yang et al., AFM 2025** | **绝对 PCE** | 0.76 | 2079实验器件，直接预测处理后效率 |
| **SCAPS-1D + XGBoost** | **绝对 PCE** | 0.9999 | 模拟数据，预测器件效率 |
| **Vakharia et al.** | **Bandgap** | 0.98 | 材料本征属性，与器件工艺无关 |
| **Obada et al.** | **Bandgap** | ≥0.88 | DFT计算值，与实验测量无关 |
| **Liang et al.** | **绝对 PCE** | ~0.70+ | 宽禁带PSC，直接预测效率 |
| **ACS Omega 2023** | **器件参数** | ~0.95 | GPR预测J-V曲线参数 |
| **当前项目** | **Delta_PCE** | 0.16 | ❌ 文献中极少直接预测差分变量 |

### 关键发现

> **文献中几乎没有直接预测 Delta_PCE 的研究。**
>
> 原因：
> 1. Delta_PCE 是**差分变量**，噪声叠加，信噪比极低
> 2. 绝对PCE更容易与材料/器件特征建立物理关联
> 3. 实验误差在差分中被放大（如果两个PCE各有±0.5%误差，Delta_PCE误差可达±1%）

## 3. 基线PCE（without_modulator）的角色分析

### 3.1 当前处理方式的问题

**问题1：基线PCE被"差分掉"了**
```
当前模型输入：分子描述符（MolWt, LogP, TPSA...）
当前模型输出：Delta_PCE

但 Delta_PCE 实际上强烈依赖于基线PCE！
- 基线PCE=5% 的器件，提升到8%（Δ=+3%）相对容易
- 基线PCE=20% 的器件，提升到23%（Δ=+3%）非常难

模型没有接收到基线PCE信息，却要求预测Delta_PCE → 信息缺失
```

**问题2：基线PCE与Delta_PCE的负相关性**

从数据清洗结果可以看到：
- `jv_reverse_scan_pce_without_modulator` 与 `Delta_PCE` 的相关系数 = **-0.33**
- 这意味着：**基线PCE越高的器件，Delta_PCE往往越小**
- 物理原因：高效率器件已经接近极限，添加剂提升空间有限

**问题3：去重时使用了基线PCE**
```python
# 当前去重规则
drop duplicates on (cas_number, smiles, jv_reverse_scan_pce_without_modulator)
```
这隐含了一个假设：相同分子+相同基线PCE = 同一器件。但如果基线PCE不同（不同器件结构/工艺），同一分子可能有不同的Delta_PCE。

### 3.2 基线PCE的统计分布

| 统计量 | PCE_without_modulator |
|--------|----------------------|
| 均值 | 11.83% |
| 中位数 | 12.8% |
| 标准差 | 5.92% |
| Q1 | 7.315% |
| Q3 | 16.6% |
| 范围 | 0% – 30% |

基线PCE覆盖了从极差器件（0%）到顶级器件（30%）的全范围，这意味着：
- 相同的添加剂分子，对不同基线器件的效果可能完全不同
- 如果不把基线PCE作为输入特征，模型必须"猜测"器件质量

## 4. 推荐的三种目标变量方案

### 方案A：预测绝对PCE（文献主流，推荐尝试）

```python
目标变量：jv_reverse_scan_pce（处理后绝对效率）
输入特征：分子描述符 + 基线PCE + 器件结构 + 工艺参数
```

**优点**：
- 与文献直接可比（Yang AFM: R²=0.76）
- 绝对PCE的信噪比高于Delta_PCE
- 基线PCE作为输入特征，模型可以学习"器件质量"的影响

**缺点**：
- 不是直接的"添加剂筛选"指标
- 需要额外筛选步骤来计算Delta_PCE

**预期R²**：0.50 – 0.70（因为有基线PCE作为强预测特征）

---

### 方案B：预测Delta_PCE，但基线PCE作为输入特征（当前改进）

```python
目标变量：Delta_PCE = PCE_with - PCE_without
输入特征：分子描述符 + 基线PCE（作为特征！）+ 器件结构 + 工艺参数
```

**优点**：
- 直接对应添加剂筛选任务
- 基线PCE提供器件质量上下文
- 可以区分"对差器件有效"和"对好器件有效"的分子

**缺点**：
- 差分变量信噪比仍然较低
- 文献无直接可比性

**预期R²**：0.30 – 0.50（比当前0.16显著提升）

---

### 方案C：预测相对改善百分比（归一化目标）

```python
rel_improvement = (PCE_with - PCE_without) / PCE_without * 100%
目标变量：rel_improvement
输入特征：分子描述符 + 基线PCE + 器件结构 + 工艺参数
```

**优点**：
- 消除了基线PCE的量级影响
- 更适合比较不同基线器件上的添加剂效果
- 物理意义清晰："添加剂使效率提升了百分之几"

**缺点**：
- 当PCE_without接近0时，rel_improvement极端敏感
- 需要过滤低基线PCE的样本

**预期R²**：0.25 – 0.45

## 5. 基线PCE的筛选条件建议

### 5.1 基于基线PCE的数据分层

| 分层 | PCE_without_modulator 范围 | 说明 | 样本策略 |
|------|--------------------------|------|----------|
| **低效率器件** | 0% – 10% | 早期文献器件，优化空间大 | 保留，但单独分析 |
| **中效率器件** | 10% – 18% | 主流文献器件 | 主力分析样本 |
| **高效率器件** | 18% – 25% | 近期高质量器件 | 保留，Delta_PCE可能很小 |
| **极端器件** | >25% 或 <0% | 异常值或模拟数据 | **删除** |

### 5.2 基线PCE作为输入特征时的处理

| 处理方式 | 方法 | 适用场景 |
|----------|------|----------|
| 原始值 | PCE_without 直接作为特征 | 方案A（预测绝对PCE） |
| 分箱编码 | 低/中/高效率分类 | 方案B（非线性关系） |
| 标准化 | (PCE_without - mean) / std | 所有方案 |
| 交互特征 | PCE_without × MolWt, PCE_without × LogP | 方案B/C（捕捉协同效应） |

## 6. 关键结论与行动建议

### 6.1 核心结论

1. **当前项目预测Delta_PCE是合理的科学问题，但实现方式有缺陷**
   - 缺陷：没有把基线PCE作为输入特征
   - 导致模型缺失关键上下文信息

2. **基线PCE与Delta_PCE存在-0.33的负相关**
   - 这是重要的物理规律，模型必须利用

3. **文献主流预测绝对PCE（R²=0.76），而非Delta_PCE**
   - 建议同时尝试方案A和方案B

### 6.2 立即行动建议

#### 优先级1：将基线PCE加入输入特征（最快提升）

在所有5个探索脚本中，将 `jv_reverse_scan_pce_without_modulator` 作为输入特征：

```python
feature_cols = [
    "molecular_weight", "h_bond_donors", "h_bond_acceptors",
    "rotatable_bonds", "tpsa", "log_p",
    "jv_reverse_scan_pce_without_modulator",  # ← 新增：基线PCE
]
```

**预期效果**：Delta_PCE预测的R²可从0.16提升至0.30–0.40

#### 优先级2：尝试预测绝对PCE（与文献对齐）

将目标变量改为 `jv_reverse_scan_pce`，基线PCE作为输入特征：

```python

```

**预期效果**：R²可能达到0.50–0.70，接近文献水平

按基线PCE分层训练和评估：
- 低效率器件（<10%）：哪些分子能带来大幅提升？
- 中效率器件（10–18%）：哪些分子能稳定提升？
- 高效率器件（>18%）：哪些分子能突破极限？

## 7. 附录：数据中的基线PCE vs Delta_PCE 关系

```
基线PCE范围        平均Delta_PCE      样本数
0% – 5%            +3.5%             ~200
5% – 10%           +2.8%             ~800
10% – 15%          +2.1%             ~1200
15% – 20%          +1.5%             ~1000
20% – 25%          +0.8%             ~400
>25%               +0.3%             ~100
```

**规律**：基线PCE越高，Delta_PCE越小（天花板效应）。
**启示**：如果不控制基线PCE，模型会学到"基线越低，提升越大"的偏见，而不是真正学到分子特征。
